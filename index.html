    Project Overview

    The Computer Lab Management System (CLMS) is designed to coordinate operations inside a university computer lab. The C++ application manages PC tracking, student reservations, session logs, and issue reporting. The implementation models real-world business constraints using professional object-oriented programming (OOP) paradigms.

         Class Diagram ("grp")
        
                        
        
                            Illustrates the object-oriented structure of CLMS, including the inheritance tree where Admin and Student derive from User, and associations for management classes.
                        
        
                        
        
                            
        
classDiagram
    class User {
        #int userID
        #string name
        #string role
        +User()
        +inputUser() void
        +displayUser()* void
    }
    class Admin {
        -string department
        +inputAdmin() void
        +displayUser() void
    }
    class Student {
        -string course
        +inputStudent() void
        +displayUser() void
    }
    class PC {
        -int pcID
        -string status
        +inputPC() void
        +displayPC() void
        +savePCToFile() void
        +viewAllPCs() void
        +searchPC() void
        +updatePC() void
        +deletePC() void
    }
    class Booking {
        -int studentID
        -int pcID
        +bookPC() void
        +viewBookings() void
    }
    class Issue {
        -int issueID
        -string description
        +reportIssue() void
        +viewIssues() void
    }
    class Session {
        +startSession() void
        +endSession() void
    }
    class Login {
        +authenticate() bool
    }
    class Dashboard {
        +displayDashboard() void
    }

    User <|-- Admin : Inheritance
    User <|-- Student : Inheritance
    main ..> User : Uses ptr
    main ..> PC : Manages
    main ..> Booking : Manages
    main ..> Issue : Manages
    main ..> Session : Session Manager
    main ..> Login : Authentication Gate
    main ..> Dashboard : Info Dashboard
                            
        
                        
        
                    
        

                    
                    
        
                        
         Sequence Diagram ("sequence")
        
                        
        
                            Traces the timeline of operations: from authentication validation to session starting, executing the booking operation (storing records in text files), and ending the session.
                        
        
                        
        
                            
        
sequenceDiagram
    autonumber
    actor User as Lab Operator
    participant M as main() Loop
    participant L as Login Class
    participant S as Session Class
    participant B as Booking Class
    participant F as Filesystem (VFS)

    User->>M: Launch Application
    M->>L: authenticate()
    L->>User: Prompt Credentials (Username, Password)
    User->>L: Enter "Fahad Mahar" / "2006"
    alt Authentication Valid
        L-->>M: return true
    else Invalid Credentials
        L-->>M: return false & terminate
    end

    M->>S: startSession()
    S->>F: Write "Session Started" to session.txt
    
    loop Operation Menu Loop
        M->>User: Display System Menu (Option 1-15)
        User->>M: Input Choice 11 (Book PC)
        M->>B: bookPC()
        B->>User: Enter Student ID & PC ID
        User->>B: Submit values (e.g. 2001, 104)
        B->>F: Append "2001 104" to booking.txt
        B-->>User: Display "Booking Successful!"
    end

    User->>M: Input Choice 15 (Exit)
    M->>S: endSession()
    S->>F: Write "Session Ended" to session.txt
    M-->>User: Print "Exiting Program..."
